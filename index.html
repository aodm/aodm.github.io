Drone Responder of Network Emergencies


Drone response to network emergencies using an Android application.

Introduction

Natural disasters can often result in network infrastructure becoming damaged or worse, completely destroyed. When such an event occurs a number of issues arise including the re-establishment of communications and networks. As seen around the world in recent times, a response to such an event has been the creation of mobile ad-hoc / mesh networks etc.

Combining drone technology with these networks would allow for greater capabilities of the networks as the drone is not bound by difficult ground terrain and as such could be utilised to assist in such a scenario.

This project will therefore focus on the response component in utilising drone technology as a solution to network disruption. This component will be focused on as it can be applied to a number of real world scenarios, for example:

Monitoring Situation
Networks utilising sensors and other technologies are often used and often in remote locations for monitoring purposes e.g. monitoring an active volcano. These networks can become disrupted for a number of reasons e.g. unstable environment, weather, animals, loss of power etc. In such a scenario, a drone responder can be deployed in order to re-establish network connectivity e.g. act as a relay node.

Emergency Response
As previously mentioned, the re-establishment of a network can be of vital importance during times of emergency for first responders, law enforcement etc. The repair of damaged infrastructure can sometimes be difficult to achieve due to a number of constraints such as location, environment, safety etc. In such a scenario, a detachment of drones can be deployed in order to provide a large geographical area with a network.

Project Goal

The goal of this project was to research and demonstrate the potential of utilising drone technology as a response to network emergencies.

An ad-hoc network represents a remote network. A monitoring node at the boundary of the ad-hoc network and the 'connected world' relays drone requests. An android application is then used to inform an end-user of the ‘Drone Request’, at which point a drone can be deployed to the requesting nodes location and then returned to its base location.

Frequently Ask Questions

The following are some general questions asked about the project.

Why this project?

I primarily chose this project because of my interest in the possible applications of drone technology. The network disruptions experienced in certain parts of the world due to natural disasters prompted me to research the potential of drone technology as a solution for such a scenario. The response to such events in recent years has been the creation of mobile networks. The network types and function vary from ad-hoc, mesh, MANET etc. The idea for the project then combined these two, the possible applications of drone technology and the use of the networks like those listed, as a response to disastrous events. Drones are not limited by difficult terrain or damaged infrastructure and as such, accompanied with the right hardware and software they can be used to assist in network establishment, range, bridging etc.

Why develop an Android Application?

Java is the main programming language thought in the (BSc) Software Development course. An android interface was available for the drone development and Android Mobile Development was also covered as an elective module during my time of studies which secured android as the platform of choice.

Why raspberry pi boards and not another?

Raspberry pi boards were chosen as a number of them were made available for this project by Cork Institute of Technology. Any of the other similar boards, Arduino etc. could also be used.

Why 3DRobotics Solo Drone?

The Solo drone was made available for this project by Cork Institute of Technology. This drone was chosen due for its technical specifications and the ability to develop for it using the dronekit SDK.
